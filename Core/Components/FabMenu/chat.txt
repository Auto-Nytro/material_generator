Yes, you can absolutely use scale!
Using scale is the best approach for performance because it triggers hardware acceleration and avoids costly layout recalculations (reflows).
## The Solution
To prevent the icon inside the Floating Action Button (FAB) from getting distorted or shrinking too much when the parent scales down, you must apply an inverse scale to the icon.
Here is how the logic works:

* FAB Active State: Scale set to 0.8 (shrinks the button).
* Icon Active State: Scale set to 1.25 (computed as 1 / 0.8) to keep its visual size intact, or adjusted slightly lower to achieve your desired smaller target size.
* Icon Switch: Swap the icon graphic at the exact midpoint of the scale transition (when opacity hits 0) for a seamless morph effect.

## CSS Example

```css
/* The FAB Base */
.fab {
  width: 56px;
  height: 56px;
  border-radius: 50%;
  transition: transform 0.3s ease;
}
/* Toggled Small State */
.fab.is-toggled {
  transform: scale(0.75); /* Shrinks overall button */
}
/* The Icon Base */
.fab-icon {
  transition: transform 0.3s ease, opacity 0.15s;
}
/* Counter-scale the icon so it doesn't look squished */
.fab.is-toggled .fab-icon {
  transform: scale(1.1); /* Adjust this value to control final icon size */
}
```

  height: number;
  iconSize: number;
  centerX?: number;
  centerY?: number;

function compileMorphTransforms(fab: ElementMetrics, close: ElementMetrics) {
  // Container Scale Factors
  const scaleX = close.width / fab.width;
  const scaleY = close.height / fab.height;

  // Icon Counter-Scale Factors (accounts for parent scaling distortion)
  const iconScaleX = (close.iconSize / fab.iconSize) * (1 / scaleX);
  const iconScaleY = (close.iconSize / fab.iconSize) * (1 / scaleY);

  const translateX = (close.centerX ?? 0) - (fab.centerX ?? 0);
  const translateY = (close.centerY ?? 0) - (fab.centerY ?? 0);

    containerTransform: `translate(${translateX}px, ${translateY}px) scale(${scaleX}, ${scaleY})`,
    iconTransform: `scale(${iconScaleX}, ${iconScaleY})`,
